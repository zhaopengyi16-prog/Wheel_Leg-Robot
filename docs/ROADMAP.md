# 路线图

## 第一阶段：仓库规则与文档骨架

- 建立根目录说明、协作规则、忽略规则和 Git LFS 规则。
- 为一级目录建立 README。
- 建立项目范围、架构、路线图、通信协议、版本兼容和系统基线文档。
- 建立实验记录模板。

## 第二阶段：版本基线整理

- 确认 STM32 工程来源、工具链版本和可编译状态。
- 确认硬件设计源文件、生产资料和 BOM 的版本基线。
- 确认机械 CAD、STL 和机器人描述包的版本基线。
- 确认 ROS2 工作区组织方式。

## 第三阶段：构建与测试流程

- 建立固件编译说明和产物输出规范。
- 建立 ROS2 构建、运行、仿真或可视化说明。
- 建立基础自动化测试和实机测试流程。
- 建立实验记录编号、归档和复盘流程。

## 第四阶段：整机集成

- 建立硬件、机械、固件和 ROS2 的兼容矩阵。
- 建立系统基线记录。
- 建立实机实验结果与版本基线的追踪关系。
