# Codex 工作规则

## 阅读顺序

1. 先阅读根目录 `README.md`，了解仓库用途、当前状态和文档索引。
2. 再阅读相关子目录的 `README.md`。
3. 如果目标目录存在 `AGENTS.md`，必须优先遵守该目录规则。
4. 修改前先确认当前文件状态，不要覆盖用户未说明的改动。

## 通用规则

- 所有无法从仓库确认的事实写“待确认”，不要猜测硬件参数、接线、版本或实验结论。
- 不要擅自移动、复制、删除旧 STM32 工程、硬件工程、机械 CAD 或 ROS2 描述包。
- 不要提交编译产物、日志、缓存、临时文件、私密配置或个人 IDE 配置。
- 不要执行 `git add`、`git commit`、`git push`、`git reset`、`git rebase`，除非用户明确要求。
- 修改源码、硬件设计源文件、机械 CAD 或实验数据前，先确认任务范围。
- 文档默认使用中文。
